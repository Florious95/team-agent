Team Agent 上线前测试清单

每次发布前逐项执行。勾选状态保存在当前浏览器的 localStorage 中，不会上传，也不依赖外部服务。

0 / 0

确认从 npm latest 安装出来的 `team-agent` binary、PATH 命中位置、runtime 目录内容完全一致。

installer

验证方法

npx @team-agent/installer@latest install
which team-agent
team-agent --version
ls -l "$(which team-agent)" ~/.local/bin/team-agent
md5 "$(which team-agent)"
md5 ~/.team-agent/runtime/team-agent

若 `which team-agent` 不是 `~/.local/bin/team-agent`，必须记录 PATH 顺序并确认实际被 shell 调用的 binary 仍是最新版本。

确认普通非 tmux 终端下 Claude Code 能被 Team Agent 正常拉起，并且不会启动即退出。

leader

验证方法

env | grep -E '^TMUX=' || true
team-agent claude
# 等待 10 秒，确认 Claude Code TUI 保持运行
# Ctrl+C 退出后检查 exit code 和相关 tmux session/log

确认 leader 向 worker 发送消息时，真实命中 pane 后不会被错误标记为 failed。

messaging

验证方法

team-agent send <worker> "请回复收到，并 report_result。"
team-agent status --json
rg -n "send.failed|matched=true|failed" .team/runtime/current

确认 shutdown 后 restart 能恢复原 worker 角色上下文，不因单个 worker 问题牵连全队 fresh spawn。

lifecycle

验证方法

team-agent shutdown --workspace . --keep-logs
team-agent restart .
team-agent status --json
team-agent send <worker> "请说明你恢复后的角色和上一轮任务上下文。"
rg -n "forced_fresh|fresh_spawn|resume" .team/runtime/current

确认线上主页可访问，核心视频素材能自动播放，中英文切换不破坏页面状态。

website

验证方法

open https://team-agent.net
curl -I https://team-agent.net
# 在桌面和移动宽度各检查一次：
# 1. 视频自动播放
# 2. 中文/英文切换
# 3. 首屏、FAQ、CTA 区域无布局破损
