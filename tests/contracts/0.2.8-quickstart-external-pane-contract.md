# 0.2.8 Quick-Start External Pane Contract

## Scope

This contract covers bug-065: quick-start / launch can locate the current tmux
client pane, but rejects an externally launched Claude Code pane because
`pane_current_command` is the official binary version string `2.1.154`.

## Contract

1. `_pane_is_usable_leader` must not reject by command name. If the pane cwd
   matches the workspace, commands such as `2.1.154`, empty string, `node`, or
   unknown binaries are usable leader panes.

2. `_resolve_leader_pane(None, provider, workspace, require_current=True)` must
   adopt the current tmux client pane when its cwd equals the workspace, with
   discovery mode `current_client`, regardless of `pane_current_command`.

3. Workspace isolation remains binding. A pane whose cwd does not equal the
   workspace must still be rejected, even if command-name gating is removed.

4. The quick-start / launch error path must not recommend a nonexistent
   `--pane` option unless quick-start actually supports that option. The
   failure text should describe the concrete mismatch, such as cwd mismatch or
   missing current tmux pane.

## Acceptance

Executable acceptance lives in
`tests/test_028_quickstart_external_pane_acceptance.py`. It uses the real
public runtime seams and tmux-pane capture format from `_legacy_pane_discovery`
instead of invented helpers.
