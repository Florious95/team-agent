# 0.2.7 Pane-As-Identity Contract

## Scope

Leader authorization is the live tmux pane identity. Command names and
leader_session_uuid are metadata, not authorization gates.

## Contract

1. Binding from the caller pane accepts any `pane_current_command`, including
   an empty string, version strings such as `2.1.154`, `node`, and unknown
   commands. The only bind refusal for caller shape is missing `$TMUX_PANE`,
   with reason `caller_pane_missing`. The legacy command-shape refusal is retired.

2. `check_team_owner` authorizes by pane equality. If the caller's live pane id
   equals `state.team_owner.pane_id`, the operation is allowed even when the
   caller's derived or inherited `leader_session_uuid` differs from the stored
   owner uuid. `_caller_identity_from_env` must use `$TMUX_PANE` when
   `TEAM_AGENT_LEADER_PANE_ID` is absent.

3. A stored owner pane that is not live must not lock out a new live caller
   pane. The liveness check must fail safe: tmux probe failure means the old
   pane is not confirmed live, so the new caller is not refused solely by the
   stale owner record.

4. On restart, stale leader receiver records from a dead tmux pane/session must
   not remain the active receiver after restart. If restart autobind can resolve
   the current caller pane, receiver state and downstream display/message
   targets use that live pane. If autobind cannot resolve a live pane, restart
   leaves the receiver unbound/rebind-required rather than falling back to the
   stale dead receiver.

5. Existing tests must not preserve old authorization semantics. Tests may
   assert that `leader_session_uuid` is stored, logged, or used for delivery
   dedupe, but not that uuid mismatch by itself refuses the current owner pane.
   Tests may assert missing `$TMUX_PANE` refusal, but not command-name
   allowlists or the legacy command-shape refusal as a valid bind refusal.

## Acceptance

The executable acceptance is `tests/test_027_pane_as_identity_acceptance.py`.
It uses the public functions `bind_owner_from_caller_pane`,
`check_team_owner`, and `restart`, plus mocked tmux/provider boundaries.
